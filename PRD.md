## 1. 项目概述

### 1.1 背景

盲盒经济在年轻群体中持续升温。用户通过购买盲盒获得随机角色/主题卡片，收集整套带来成就感。本产品是一款移动端优先的虚拟盲盒抽卡应用，用户可在多个系列中抽取虚拟卡片、管理收藏、分享成果。

### 1.2 目标

1. **第一性原理**：让用户获得"开盒那一刻"的惊喜感和收集成就感
2. **产品目标**：提供流畅、美观、沉浸式的虚拟盲盒体验
3. **阶段目标**：MVP 完成核心抽卡 + 收藏闭环，日活用户停留 > 5 分钟

### 1.3 范围

**包含（MVP）：**
- 系列浏览与选择
- 虚拟盲盒抽取（含 3D 开盒动画）
- 抽卡结果展示
- 收藏管理（按稀有度筛选）
- 分享卡片生成

**不包含（后续迭代）：**
- 用户注册/登录体系
- 付费充值体系
- 社交/好友系统
- 交易/交换市场
- 后台管理系统
- 真实支付对接

## 2. 用户角色

| 角色 | 说明 | 核心行为 |
|------|------|----------|
| 普通用户 | 未登录、无持久化身份的用户 | 浏览系列 → 抽取盲盒 → 查看收藏 → 分享卡片 |

MVP 阶段暂不做用户系统和差异化角色，所有用户视为匿名访客，使用 localStorage 或 IndexedDB 做本地持久化。

## 3. 功能模块

### 3.1 首页 - 系列列表

**页面组成：**

| 组件 | 说明 |
|------|------|
| 顶部导航栏 | 左：品牌名"盲盒"，右：搜索图标（MVP 占位，暂不可用） |
| 标题"探索系列" | 区域标题 |
| 系列网格 | 2 列网格布局，每项为 SeriesCard |
| 收藏进度入口 | 底部卡片：显示"我的收藏" + 进度条 + 数字（已收集/总数） |

**SeriesCard 组件：**
- 封面图区域：3:4 比例，浅色占位背景，展示系列 emoji/图标
- 信息区：系列名称 + 款数（"N 款"）

**交互行为：**
- 点击 SeriesCard → 跳转抽卡页，传入系列 ID
- 点击收藏进度入口 → 跳转收藏页
- 搜索按钮：MVP 暂不做搜索逻辑，点击无响应或用 Toast 提示"即将上线"

**边界情况：**
- 系列列表为空 → 显示空状态："暂无系列，敬请期待"
- 系列名称过长 → 文字截断省略（最多 2 行）
- 系列数量为奇数 → 最后一张卡片居左

### 3.2 抽卡页 - 盲盒抽取

**页面组成：**

| 组件 | 说明 |
|------|------|
| 顶部栏 | 左：关闭按钮（✕），右：显示余额 |
| 3D 盒子区域 | CSS 3D 透视盒，磨砂白哑光质感 |
| 提示文字 | 初始为"点击盒子开箱"，抽卡动画中变化 |
| 抽卡按钮 | "抽一次"主按钮 |
| 保底提示 | 底部："保底：已抽 N 次" |

**交互行为：**
1. **点击盒子** 或 **点击抽卡按钮** → 触发抽卡
2. 抽卡流程：
   - 关闭按钮变为不可点击
   - 盒子触发开盒动画（2.5 秒）
   - 提示文字变为"开盒中..."
   - 动画结束后自动跳转至结果页
3. 点击关闭按钮 → 返回首页（确认弹窗："确定退出抽卡？本次机会不消耗"）
4. 余额不足时 → 按钮置灰，提示"余额不足"

**3D 盒子规格：**
- 尺寸：210×210px
- 6 个 CSS 面构建，带透视变换
- 默认旋转：rotateX(-8deg) rotateY(-20deg)
- 开盒动画：盒盖翻起 + 内容物发光涌现，持续 2.5 秒
- 质感：纯白背面加极淡阴影，无纹理磨砂

**边界情况：**
- 快速连点 → 锁定状态，动画期间不可再次点击
- 所有卡片已集齐 → 弹出提示"恭喜！你已经收集齐该系列所有卡片！"
- 无网络 → 不影响 Mock 模式

### 3.3 抽卡结果页

**页面组成：**

| 组件 | 说明 |
|------|------|
| 顶部栏 | 关闭按钮（✕），无标题 |
| 卡片展示 | 模拟实物卡片，含稀有度徽标、艺术图、名称、系列名 |
| 卡片详情名称行 | 全名：如"星海旅人 · 极光" |
| 稀有度文字 | 中文名称：如"稀有度：传说" |
| 操作按钮组 | "查看详情"（主色） + "分享"（次色） |

**卡片视觉规格：**
- 尺寸：200×280px
- 圆角：14px
- 边框：稀有度对应色的 2px 边框
- 背景：渐变底色（依稀有度调整深浅）
- 带内发光的柔和阴影

**交互行为：**
- "查看详情" → 展示更多卡片信息（适用场景：卡片介绍、故事背景等，MVP 可用 Modal 承载）
- "分享" → 跳转分享页
- 关闭按钮 → 回到当前系列的抽卡页

**边界情况：**
- Secret 稀有度 → 额外特效（粒子闪光/微动效），边框使用渐变金
- 点击"查看详情"但卡片无详情内容 → 显示"暂无介绍"

### 3.4 收藏页

**页面组成：**

| 组件 | 说明 |
|------|------|
| 顶部导航栏 | 左：返回按钮，中："我的收藏" |
| 胶囊分段筛选 | 横向滚动的胶囊按钮：全部 / Common / Rare / Epic / Legendary |
| 收藏网格 | 3 列网格，每项为 CollectionItem |
| 统计信息栏 | 底部：已收集 N / M，🏆 N% |

**CollectionItem 组件：**
- 1:1 正方形卡片
- 已收集：展示对应 emoji/图标，底部稀有度色条
- 未收集：浅色虚线框，显示"?"占位

**稀有度色条：**
| 稀有度 | 色条颜色 |
|--------|----------|
| Common | 暖灰麻布 #D4C9B8 |
| Rare | 柔灰蓝 #9FB8D4 |
| Epic | 薰衣草紫 #B8A8D8 |
| Legendary | 暖金 #E8C87A |
| Secret | 浅金渐变 |

**交互行为：**
- 点击已收集项 → 查看该卡片的详情（跳转详情弹窗或独立详情页，MVP 用弹窗）
- 点击未收集项 → 无响应或小提示"继续抽卡获取"
- 筛选切换 → 仅显示对应稀有度的卡片，未收集项保留
- 统计信息根据当前筛选动态更新

**边界情况：**
- 某稀有度下无任何卡片 → 显示"该稀有度暂无卡片"
- 全部收集完成 → 统计显示"已收集 全 / N"，🏆 100%，出现庆祝动效
- 筛选到无未收集项的稀有度 → 网格全部为实卡，无占位

### 3.5 分享页

**页面组成：**

| 组件 | 说明 |
|------|------|
| 顶部导航栏 | 左：返回按钮，中："分享卡片" |
| 预览卡片 | 纯白背景卡片，含艺术图、名称、系列、稀有度、用户名 |
| 保存按钮 | "↓ 保存图片" |
| 分享按钮 | "↗ 分享到..." |

**预览卡片规格：**
- 最大宽度 320px
- 纯白背景，16px 圆角
- 顶部视觉区：大艺术图(120×120) + 名称 + 系列
- 底部信息栏：稀有度 + @username，浅灰分割线分隔
- 卡片整体阴影柔和

**交互行为：**
- "保存图片" → 将卡片转为图片并保存到设备相册（使用 html2canvas 或类似方案）
- "分享到..." → 调用 Web Share API（优先）或复制链接兜底
- 返回按钮 → 回到抽卡结果页

**边界情况：**
- Web Share API 不可用（非 HTTPS 或不受支持浏览器）→ 回退到"复制链接"按钮
- 保存图片权限不足 → 提示"保存失败，请检查相册权限"
- 用户名未设置 → 显示默认占位"@神秘收藏家"

## 4. 数据模型

### 4.1 Serie（系列）

```typescript
interface Serie {
  id: string;               // 唯一标识，如 "forest-tales"
  name: string;             // 系列名称，如 "森林物语"
  emoji: string;            // 封面 emoji，如 "🌿"
  description: string;      // 系列简介
  totalCards: number;       // 总卡数，如 12
  cards: BlindBox[];        // 该系列包含的所有卡片
  createdAt: string;        // ISO 日期字符串
}
```

### 4.2 BlindBox（卡片）

```typescript
interface BlindBox {
  id: string;               // 唯一标识，如 "forest-tales-01"
  serieId: string;          // 所属系列 ID
  name: string;             // 卡片名称，如 "星海旅人"
  subtitle?: string;        // 副标题/变体，如 "极光"
  emoji: string;            // 展示 emoji
  rarity: Rarity;           // 稀有度枚举
  description?: string;     // 卡片故事/介绍
  seriesIndex: number;      // 在该系列中的编号（1-based）
}

type Rarity = 'common' | 'rare' | 'epic' | 'legendary' | 'secret';
```

### 4.3 Collection（用户收藏）

```typescript
interface Collection {
  cardId: string;           // 卡片 ID
  serieId: string;          // 系列 ID
  acquiredAt: string;       // 获取时间 ISO 字符串
  count: number;            // 重复数量（抽到已拥有卡片时累加）
}

// 全局收藏状态（用户级）
interface UserCollectionState {
  collections: Collection[];           // 已收集的所有卡片
  drawCounters: Record<string, number>;// 各系列已抽次数（用于保底）
}
```

## 5. 稀有度规则

### 5.1 五级稀有度体系

| 稀有度 | 标识 | 边框色 | 底色 | 展示优先级 |
|--------|------|--------|------|-----------|
| Common（普通） | 暖灰麻布 | #D4C9B8 | #FAFAF7 | 5（最弱） |
| Rare（稀有） | 柔灰蓝 | #9FB8D4 | #F5F8FC | 4 |
| Epic（史诗） | 薰衣草紫 | #B8A8D8 | #F8F5FC | 3 |
| Legendary（传说） | 暖金 | #E8C87A | #FFFBF5 | 2 |
| Secret（隐藏） | 浅金渐变 | 渐变金线 | #FFFDF8 | 1（最强） |

### 5.2 概率权重（推荐）

| 稀有度 | 概率 | 说明 |
|--------|------|------|
| Common | 55% | 最常见的卡片 |
| Rare | 25% | 较常见 |
| Epic | 12% | 中等稀有 |
| Legendary | 7% | 稀有的传说级 |
| Secret | 1% | 隐藏款，最稀有 |

### 5.3 保底机制

- **硬保底**：每抽 60 次未出 Legendary，第 60 次强制出 Legendary
- **软保底**：40 抽后 Legendary 概率逐步提升（每抽 +3%）
- Secret 不计入保底（始终 1%）
- 保底计数器按系列独立计算
- 连续抽到 3 张相同的 Common → 下张 Common 权重转移到 Rare+

## 6. UI/UX 规范

### 6.1 布局原则

| 原则 | 说明 |
|------|------|
| 移动端优先 | 以 390px 宽度为设计基准，渐进增强到桌面 |
| 大量留白 | 卡片间距 ≥ 12px，内边距 ≥ 16px |
| 柔和圆角 | 大组件 24px，卡片 12px，按钮 10px |
| 阅读层级 | 标题 22px/600w，正文 15px/400w，辅助 13px |

### 6.2 配色体系

| 用途 | 色值 | 名称 |
|------|------|------|
| 页面底色 | #F8F9FA | 极浅灰白 |
| 主色调 | #A8C5D6 | 雾蓝（标题栏、主按钮、进度条） |
| 辅色 1 | 暖杏（#E8C87A → #F5E6C8 区段） | 暖色调点缀 |
| 辅色 2 | 鼠尾草绿（#B8C9B8 → #E8F0E8 区段） | 自然感点缀 |
| 文字主色 | #1C1C1E | 深黑灰 |
| 文字辅色 | #8E8E93 | 中灰 |
| 文字弱色 | #C7C7CC | 浅灰 |
| 分割线 | #F2F3F5 | 极浅灰 |

### 6.3 动效规范

| 场景 | 时长 | 描述 |
|------|------|------|
| 开盒动画 | 2.5 秒 | 盒盖翻起 + 内容物发光涌现 |
| 页面跳转 | 300ms | 淡入/上滑 |
| 按钮反馈 | 150ms | 微缩放（1→0.97→1） |
| Secret 出场 | 2.0 秒 | 粒子闪光 + 渐显 + 边框发光脉冲 |

### 6.4 交互反馈

- 所有可点击元素需有 :active 态反馈（缩放或颜色变化）
- 加载状态：骨架屏优先（避免菊花转）
- 无数据状态：展示友好空态插画 + 引导文案
- Toast 提示统一使用底部轻提示，持续 2 秒自动消失

## 7. 技术方案建议

### 7.1 框架选型

| 层 | 推荐方案 | 理由 |
|----|----------|------|
| 框架 | React 18 + Vite | 轻量、快速 HMR、生态成熟 |
| 路由 | React Router v6 | SPA 路由，支持 HashRouter（静态部署友好） |
| 状态管理 | React Context + useReducer | MVP 阶段无需引入 Redux/Zustand |
| 样式方案 | CSS Modules 或 Tailwind CSS | 推荐 CSS Modules 保持零运行时、高可维护性 |
| 动画 | CSS Animation + Framer Motion | 3D 盒子用 CSS 3D transform，出场动效用 Framer |
| 3D 盒子 | CSS perspective + transform | 无需 Three.js，CSS 3D 足够表现磨砂盒质感 |
